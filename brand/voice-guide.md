# 旁白語氣指南 — voice-guide

> WF3（腳本生成）的 agent 必讀。定義「我們怎麼說話」與雙稿格式。
> **目前階段：採用有質感的 AI 配音（TTS）為主**，輸出重點放在 `script-tts.md`。
>
> **指定聲線（Pilot 調查員）：沉穩、磁性的成熟男聲，有抑揚頓挫。**
> - 音色：中低音、胸腔共鳴、帶磁性；像深夜紀錄片或懸疑 podcast 的旁白者。
> - 節奏：語速偏慢（約 0.9x）；**重點在抑揚頓挫**——關鍵字放慢加重、懸念處壓低、轉折處提速，靠語調起伏而非音量製造張力。
> - 情緒：克制、不戲劇化、不上揚討好；句尾多為下沉的肯定句。
> - 段落間留足停頓（800ms–1.5s），讓資訊沉澱。
>
> **TTS 引擎與聲音建議**：
> - Demo / 免費：**Edge 神經語音**（本專案 `tools/tts.mjs` 已整合，免金鑰）。男聲首選 `zh-CN-YunjianNeural`（磁性、適合懸疑紀錄片）；備選 `zh-CN-YunxiNeural`、`zh-TW-YunJheNeural`（繁中男聲）。
> - 正式 / 進階：VoAI、Fish Audio、Azure Speech（神經男聲 + 自訂 SSML）。
>
> **抑揚頓挫的 SSML 寫法**（`script-tts.md` 用）：以 `<prosody rate="-12%" pitch="-2st">` 設定整體沉穩基調；關鍵句用 `<prosody rate="-20%"><emphasis level="strong">…</emphasis></prosody>`；停頓用 `<break time="1s"/>`；懸念處 `<prosody pitch="-3st" volume="-2dB">…</prosody>`。

## 整體聲音
- 基調：沉穩、理性、帶懸念但**不聳動**。像一位冷靜的調查者在跟你還原案情。
- 人稱：第二人稱拉近距離（「你可以想像……」）+ 中性旁白陳述。避免第一人稱情緒宣洩。
- 對受害者：克制、尊重。對嫌疑人：嚴守「涉嫌」用語（見 legal-redlines）。

## 用詞規範
- 短句為主，長短交錯製造節奏。一句話講一件事。
- 善用設問製造懸念：「但問題來了——」「如果是這樣，那為什麼……？」
- 章節間用過渡句承接，不要硬切。
- 避免：網路爛梗、過度形容詞、廉價驚嘆、對未定讞案件的定罪式斷言。

## 結構公式（每集）
0. **解說員開場 [HOST-01]**：自我介紹＋簡述背景，結尾以「現在讓我們回到 <案發年份> 年那個 <季節>」帶入（年份照案件實際填，19XX 或 20XX 皆可；季節依案件，未必是冬天）。
0.5 **黑屏打字機日期卡 [INTRO]**：日期＋地點逐字浮現（打字機聲），再進正片。
1. **冷開場鉤子**（0–60 秒）：丟出最反常的一個謎團/畫面，不先講背景。
2. **案件背景**：人事時地物，建立座標。
3. **事件經過**：按時間線推進，埋線索。
4. **調查與爭議**：警方/司法歷程、各方理論、矛盾點。
5. **推理收束**：攤開線索，帶觀眾推理；明確區分「已知事實」與「合理推測」。
6. **結尾 [HOST-結尾]**：案件現狀（已定讞/上訴中/未解）+ 對觀眾的一句留白。
7. **片尾 [ENDING]**：一句懸而未決的問句 → 真實案件圖片／庭審影片 → 電影謝幕字卡。**收尾 BGM**：結尾詞時先小聲墊底，講完進謝幕段轉強至正常音量（make-demo 自動處理）。

## 懸疑與反轉技法（★核心，每集必用）

懸案頻道的命脈是「懸念」與「反轉」。每一段都要讓觀眾想知道「然後呢？」。

**鉤子與懸念**
- 冷開場給「畫面＋謎團」，不給結論：先丟最反常的一幕，再慢慢回頭。
- 每段結尾留鉤子或設問：「但問題來了——」「沒有人想到，這只是開始。」「然而，真相比這更黑暗。」把觀眾拉進下一段。
- 第二人稱代入：「如果是你，你會發現嗎？」

**延遲揭露（最重要）**
- 關鍵事實不要一次講完：先給線索與矛盾、製造猜測，再在情緒高點揭露。
- 鋪「你以為……」再翻「但其實……」。例：先讓觀眾以為兇手是陌生人，再揭露他們是受害者的「朋友」。

**反轉素材怎麼找**
- 從 factcheck/dossier 挑「反常識」的真實事實當反轉點：看似普通的兇手、被社會忽視的受害者、意想不到的動機（如冒領福利）、技術突破翻案、官方失誤。
- 每集至少 2–3 個明確「翻轉時刻」，用節奏與停頓放大（[停頓] 後再丟關鍵句）。

**節奏**
- 短句堆疊製造緊迫；長句鋪陳後用一記短句收殺（[加重]）。
- 揭露前壓低、放慢；揭露瞬間停頓再給。
- 不要平鋪直敘「先A再B」；要「為什麼會這樣？→線索→但矛盾→真相」。

**禁忌**
- 不在開頭劇透結局；不寫平淡總結式旁白（每句都要推進懸念或情緒）；反轉必須建立在 factcheck 已證實事實上，不得虛構。

## 雙稿規格（本頻道同時產兩種稿）

### A. 自然稿 `script-natural.md`（給真人配音／你自己錄）
- 純口語，讀起來順。
- 用方括號標演播提示，不念出來：`[停頓]` `[加重]` `[放慢]` `[語氣轉折]` `[壓低聲音]`
- 段落對應分鏡，每段標小標題方便錄音定位。

範例：
```
[壓低聲音] 一九九X年的那個冬夜，沒有人聽見任何聲響。[停頓]
但隔天早上，整棟公寓——空了。[加重]
```

### B. TTS 稿 `script-tts.md`（給 VoAI / Fish Audio 等）
- 同內容，但把演播提示轉成 TTS 可用標記。
- 標記慣例（純文字、與引擎無關，使用時再轉成該引擎語法）：
  - 停頓：`<break time="800ms"/>`
  - 語速：`<prosody rate="slow"> … </prosody>`
  - 重音/強調：`<emphasis> … </emphasis>`
- 句子盡量斷乾淨，避免破音；數字、英文、年份標註讀法（如「1997」→「一九九七年」）。

範例：
```
<prosody rate="slow">一九九七年的那個冬夜，沒有人聽見任何聲響。</prosody><break time="800ms"/>
但隔天早上，整棟公寓，<emphasis>空了</emphasis>。
```
