# 移機指南 — Miao Channel 懸案推理頻道專案

> 把整套生產線搬到另一台電腦。重點：**有兩個資料夾要一起搬，且必須維持「同層」關係**，還有幾個東西不能直接複製（要在新機重建）。

---

## ◆ 0. 先理解結構（最重要）

```
任意磁碟\Claude profect\          ← 父資料夾（名字可不同，但底下兩個要同層）
├── Miao Channel\                 ← 專案本體（8.4 GB）
│   ├── tools\        渲染/工作流腳本（零 npm 依賴）
│   ├── cases\        各案研究/腳本/製作包
│   ├── assets\       真實素材庫 real-library（1.2 GB）
│   ├── web\media\    成片 + history（3.5 GB）
│   ├── episodes\     集數索引（★連結，搬機會斷，要重建）
│   ├── tools\build\  渲染快取（★2.5 GB，可不搬、會自動重生）
│   └── .env          ★API 金鑰（PEXELS/YOUTUBE）——跟著搬就好
└── voice-engine\                 ← ★語音引擎，獨立同層資料夾（9.9 GB）
    ├── .hf-cache\    VoxCPM2 模型權重（4.6 GB，搬了省下載）
    └── .venv\        Python 虛擬環境（★5.2 GB，不可複製，要在新機重建）
```

**鐵則：`Miao Channel` 與 `voice-engine` 必須放在同一個父資料夾底下、互為兄弟。** make-demo 用 `專案上一層/voice-engine` 找語音引擎。磁碟代號、父資料夾名稱都可變，只要這個相對關係不變。

## ◆ 1. 複製（在舊機執行）

用 robocopy 排除「會重生／不可攜」的部分（省約 13 GB）：

```powershell
robocopy "D:\Claude profect" "E:\移機\Claude profect" /E /XJD `
  /XD "episodes" "build" ".venv" "__pycache__" "node_modules" ".git"
```
- `/XJD`：跳過連結點（避免沿 episodes 連結重複複製）
- 排除 `build`(渲染快取)、`.venv`(虛擬環境)、`episodes`(連結)、`__pycache__`
- **保留** `.hf-cache`（模型權重，搬了新機不必重抓 4.6 GB）、`.env`（金鑰）

把 `E:\移機\Claude profect\` 整包搬到新機，例如放到 `D:\Claude profect\`。

---

## ◆ 2. 新機安裝相依

1. **NVIDIA GPU + 最新驅動/CUDA**（語音克隆、CLIP 配對都吃 GPU；沒有 GPU 跑不動 VoxCPM）
2. **Node.js**（LTS）— 跑 tools（零 npm 依賴，裝 node 即可）
3. **ffmpeg / ffprobe** — 下載後加入系統 PATH（`ffmpeg -version` 要能跑）
4. **Python 3.11 + py 啟動器**（安裝時勾「Add python.exe to PATH」「py launcher」）
5. **yt-dlp**：`py -m pip install -U yt-dlp`（YouTube 真實影片用）
6. **繁中字型**：make-demo 用 `C:\Windows\Fonts\msjh.ttc`（微軟正黑）。Windows 繁中版內建；若無，安裝正黑體或改 make-demo 字型路徑。

## ◆ 3. 重建 voice-engine 虛擬環境（venv 不可複製）

```powershell
cd "D:\Claude profect\voice-engine"
py -m venv .venv
.\.venv\Scripts\python -m pip install -U pip
# 依新機 CUDA 版本裝 torch（範例 cu121）：
.\.venv\Scripts\python -m pip install torch --index-url https://download.pytorch.org/whl/cu121
# VoxCPM2 及相依（依 openbmb/VoxCPM2 官方 README）：
.\.venv\Scripts\python -m pip install voxcpm transformers accelerate soundfile numpy scipy einops safetensors
```
- 模型 `openbmb/VoxCPM2` 權重已隨 `.hf-cache` 搬來，不會重抓。
- CLIP 配對也用這個 venv（make-demo 走 `voice-engine\.venv\Scripts\python.exe`＋`tools\clip-score.py`），需 torch+transformers——上面已涵蓋。

啟動常駐語音伺服器測試：`& "D:\Claude profect\voice-engine\start-server.ps1"`（首次載入 VoxCPM2 數十秒）。

## ◆ 4. 重建 episodes 集數索引（連結搬機會斷）

在新機 PowerShell 跑（免管理員；路徑改成新機的）：

```powershell
$root="D:\Claude profect\Miao Channel"; $epDir=Join-Path $root "episodes"
New-Item -ItemType Directory -Force $epDir | Out-Null
$eps=[ordered]@{ 'ep001-snowtown-murders'='snowtown-murders';'ep002-golden-state-killer'='golden-state-killer';'ep003-peter-kurten'='peter-kurten';'ep004-dana-ireland'='dana-ireland';'ep005-stephanie-lazarus'='stephanie-lazarus';'ep006-esther-gonzalez-45'='esther-gonzalez-45' }
foreach($ep in $eps.Keys){ $slug=$eps[$ep]; $d=Join-Path $epDir $ep; New-Item -ItemType Directory -Force $d|Out-Null
  $case="$root\cases\$slug"; if(Test-Path $case){ New-Item -ItemType Junction -Path "$d\案件檔(腳本+製作包+研究)" -Target $case -ErrorAction SilentlyContinue|Out-Null }
  $lib="$root\assets\$slug\real-library"; if(Test-Path $lib){ New-Item -ItemType Junction -Path "$d\真實素材庫" -Target $lib -ErrorAction SilentlyContinue|Out-Null }
  $v="$root\web\media\$slug-demo.mp4"; if(-not(Test-Path $v)){$v="$root\web\media\history\$slug-demo.mp4"}; if(Test-Path $v){ New-Item -ItemType HardLink -Path "$d\成片.mp4" -Target $v -ErrorAction SilentlyContinue|Out-Null }
}
"episodes 重建完成"
```

## ◆ 5. 驗證（新機）

```powershell
cd "D:\Claude profect\Miao Channel"
node -e "import('./tools/load-env.mjs').then(()=>console.log('PEXELS',!!process.env.PEXELS_KEY,'YOUTUBE',!!process.env.YOUTUBE_API_KEY))"   # 應 true true
py -m yt_dlp --version          # yt-dlp 可用
ffmpeg -version                 # ffmpeg 可用
& ".\..\voice-engine\start-server.ps1"   # 語音伺服器起得來
# 小測試渲染（先把某集 episode.json 截成 3-4 段，或直接全片）：
node tools/build-episode.mjs --slug golden-state-killer
node tools/make-demo.mjs --slug golden-state-killer
```
成片出現在 `web\media\golden-state-killer-demo.mp4` 即成功。**渲染需真實網路**（抓 YouTube/Pexels/Commons），新機用一般終端機跑即可（會繼承 .env 與系統環境）。

## ◆ 6. AI 記憶延續（選配）

Claude 的跨 session 記憶在**使用者設定檔**裡、不在專案內：
`C:\Users\<你>\.claude\projects\d--Claude-profect-Miao-Channel\memory\`
要讓新機的 AI 無縫接上，把這個 `memory\` 資料夾也複製到新機對應位置。核心規則另已寫進 [AI接手指令.md](AI接手指令.md)，把那份貼給新機 AI 即可快速接手。
